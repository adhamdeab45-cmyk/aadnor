# QA — ADNOR V302 Firebase Only

تم تنفيذ الفحوص التالية قبل ضغط النسخة:

- فحص تركيب جميع ملفات JavaScript وService Worker: ناجح.
- فحص JSON لقواعد قاعدة البيانات والتخزين وملفات الإعداد: ناجح.
- فحص روابط الملفات المحلية ومعرّفات HTML المكررة: ناجح.
- مطابقة جميع العمليات المطلوبة في صفحات المستخدم والأدمن والوكيل مع طبقة Firebase Direct: ناجح، ولا توجد عملية ناقصة.
- اختبار محاكى كامل للعمليات الأساسية: إنشاء مستخدم، شراء تذاكر، العجلة، المكافأة اليومية، طلب إيداع، طلب سحب، الدعم، قبول الإيداع، تعديل الرصيد، إنشاء وكيل، فحص المستخدم وشحنه: ناجح.
- إضافة قيم احتياطية صحيحة للجوائز إذا كانت قاعدة البيانات فارغة أو تحتوي قيمة صفرية: يومي 4,500، أسبوعي 15,000، شهري 45,000، سنوي 750,000 دولار.
- انتهاء VIP يُطبّق عند تسجيل الدخول إذا انتهى موعده.
- لا يوجد استيراد Firebase Functions SDK.
- لا يوجد `package.json` أو `package-lock.json` في جذر المشروع.
- لا يوجد مجلد `functions` ولا أمر npm أو Node build.
- Service Worker يحمل رقم Cache جديد ويشمل طبقة Firebase Only.
- سوق ADN وكوده غير موجودين.

## حدود الفحص

لم يتم إجراء كتابة حية على مشروع Firebase الحقيقي لأن ذلك يحتاج تسجيل دخولك وقواعد المشروع المنشورة. بعد الرفع يجب نشر ملفي قواعد Realtime Database وStorage المرفقين، ثم تجربة دورة واحدة بمبالغ تجريبية قبل الاستخدام الفعلي.
