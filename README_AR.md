# ADNOR V302 — Firebase Only

هذه النسخة مصممة خصيصاً لوضع المشروع الحالي:

- الاستضافة: Render Static Site
- تسجيل الدخول: Firebase Authentication
- البيانات: Firebase Realtime Database
- إثباتات الإيداع: Firebase Storage
- لا تحتاج Cloud Functions ولا Blaze ولا سيرفر خلفي

## الملفات التي ترفعها إلى GitHub

ارفع محتويات هذا المجلد كلها مباشرةً في جذر المستودع. يجب أن يظهر `index.html` مباشرة في الصفحة الأولى من GitHub، وليس داخل مجلد إضافي.

لا يوجد في النسخة `package.json` ولا مجلد `functions`، لذلك Render لن يشغّل `npm install`.

## خطوة Firebase الضرورية

الواجهة تعمل مباشرةً مع قاعدة البيانات، لذلك قواعد Realtime Database الجديدة ضرورية.

1. افتح Firebase Console.
2. افتح مشروع `adhamnnn-8a4d2`.
3. ادخل Realtime Database ثم Rules.
4. انسخ محتوى `FIREBASE_DATABASE_RULES_COPY.json` كاملاً.
5. اضغط Publish.
6. افتح Storage ثم Rules.
7. انسخ محتوى `FIREBASE_STORAGE_RULES_COPY.txt` واضغط Publish.

لا تستورد `database.seed.json` فوق قاعدة موجودة قبل أخذ نسخة احتياطية. الملف مخصص لمشروع جديد أو لاستعادة الإعدادات الأساسية فقط.

## إنشاء وكيل

فعّل Email/Password من Firebase Authentication → Sign-in method.

الأدمن يستطيع إنشاء وكيل جديد باسم دخول وكلمة سر من لوحة الإدارة. عند إنشاء وكيل جديد يتم إنشاء حساب Firebase Authentication ثانوي دون إخراج الأدمن من حسابه.

في نسخة Firebase Only لا يمكن للأدمن تغيير كلمة سر وكيل موجود أو حذف حساب Authentication الخاص به من المتصفح. يستطيع إيقاف صلاحية الوكيل أو حذف بيانات دوره، أما تغيير كلمة السر أو حذف حساب Auth فيتم من Firebase Console.

## ملاحظة صريحة عن الأمان

لأن هذه النسخة لا تستخدم سيرفراً، فإن منطق التذاكر والألعاب والأرصدة يعمل داخل المتصفح ومتصل مباشرةً بـFirebase. القواعد تحد الصلاحيات وتمنع كثيراً من الوصول غير المسموح، لكنها لا تعطي حماية مالية مماثلة لسيرفر موثوق.

استخدم النسخة للتطوير والتجربة والتشغيل اليدوي تحت مراجعة الأدمن. قبل استقبال أموال حقيقية أو إطلاق عالمي، يجب نقل العمليات المالية الحساسة إلى Cloud Functions أو سيرفر خاص.
