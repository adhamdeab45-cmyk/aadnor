# أمان ADNOR V300

## قواعد لا تقبل الاستثناء

- لا تضع ملف حساب خدمة أو مفتاح `ADMIN_BOOTSTRAP_KEY` داخل الاستضافة أو GitHub.
- لا تسمح لأي صفحة متصفح بتعديل الأرصدة أو التذاكر أو نتائج السحب مباشرةً.
- كل عملية مالية في هذه النسخة تمر عبر Cloud Functions وتتحقق من حالة الحساب والصلاحية الحية.
- بعد منح أو سحب صلاحية أدمن/وكيل، يجب تسجيل الدخول من جديد؛ الدوال تسحب الجلسات القديمة أيضاً.
- إثباتات الدفع غير قابلة للتعديل أو الحذف بواسطة المستخدم بعد رفعها.
- احتفظ بنسخة احتياطية قبل تشغيل seed أو migration أو نشر قواعد جديدة.

## قبل استقبال أموال حقيقية

1. انشر الدوال والقواعد على مشروع اختبار أولاً.
2. شغّل سيناريوهات الإيداع والسحب والشحن والتذاكر والنتائج برصيد اختباري.
3. اجعل طرق الدفع غير مفعلة إلى أن تدخل بيانات حسابات حقيقية وتراجعها.
4. اربط مزود الدفع الآلي من خلال Webhook موثق من المزود، وليس من JavaScript في الواجهة.
5. فعّل تنبيهات الميزانية والسجلات في Google Cloud/Firebase.
6. راجع قوانين السحوبات والألعاب والتحقق من العمر والهوية في كل دولة مستهدفة.

## الإبلاغ عن ثغرة

استخدم بريد الدعم الحقيقي الذي تضيفه الإدارة داخل إعدادات الموقع. لا تنشر تفاصيل الثغرات أو مفاتيح الوصول علناً.
